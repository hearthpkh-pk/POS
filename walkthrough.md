# บันทึกสรุปการปรับปรุงระบบจัดการเมนูและระบบตัวเลือกอาหาร (Menu Variants & Option Selector Walkthrough)

ผมได้ปรับเปลี่ยนสถาปัตยกรรมระบบเพื่อรองรับ **"ตัวเลือกภายในเมนู" (Menu Variants)** เช่น การเลือกเนื้อสัตว์ (ไก่, หมู, เนื้อ) ที่มีราคาแตกต่างกันตามที่คุณระบุเรียบร้อยแล้วครับ โดยไม่ต้องแยกบันทึกเป็นเมนูใหม่ให้ซ้ำซ้อนหน้าร้าน

---

## 🛠️ รายละเอียดการแก้ไขและฟีเจอร์ที่พัฒนาเพิ่ม (Key Implementations)

### 1. โครงสร้างข้อมูลแบบตัวเลือกในฐานข้อมูล (Database Schema Upgrade)
*   **สร้างไฟล์ Migration:** [00017_add_menu_item_variants.sql](file:///c:/dev/POS/supabase/migrations/00017_add_menu_item_variants.sql) สำหรับเพิ่มคอลัมน์ `variants` ชนิดข้อมูล `JSONB` ในตาราง `menu_items`
*   **รูปแบบการจัดเก็บตัวเลือก:**
    ```json
    [
      {"name_th": "ไก่", "name_en": "Chicken", "price_cents": 15900},
      {"name_th": "หมู", "name_en": "Pork", "price_cents": 17900},
      {"name_th": "เนื้อ", "name_en": "Beef", "price_cents": 18900}
    ]
    ```

### 2. ป๊อปอัพเลือกตัวเลือกสไตล์ iOS (POS Variant Selection Modal)
*   **สร้างคอมโพเนนต์ใหม่:** [POSVariantModal.js](file:///c:/dev/POS/src/js/views/pos/modals/POSVariantModal.js) เป็นโมดอลแบบ iOS Premium Card
*   เมื่อคลิกเลือกสินค้าที่มีตัวเลือกย่อย (เช่น ข้าวผัด, เบอร์เกอร์, เคซาดิญ่า, ทาโก้) หน้าร้าน POS ระบบจะเปิดโมดอลเพื่อให้เลือกตัวเลือกร่วมกับราคาพิเศษทันที ก่อนจะแอดสินค้าเข้าตะกร้าอย่างราบรื่น

### 3. การคำนวณและประมวลผลระบบตะกร้าและประวัติบิล (Cart Integration)
*   ปรับปรุง [POSView.js](file:///c:/dev/POS/src/js/views/POSView.js) และ [POSService.js](file:///c:/dev/POS/src/js/services/POSService.js)
*   สินค้าในตะกร้าจะถูกจับคู่และแยกชิ้นด้วยรหัสผสม (เช่น `item_id-Beef` หรือ `item_id-Pork`) เพื่อคำนวณจำนวนชิ้นและผลรวมทางการเงินในตะกร้าได้อย่างถูกต้องแยกจากกัน
*   **การบันทึกบิล:** สคริปต์สั่งจ่ายเงินและบันทึกดราฟต์จะทำความสะอาดลบส่วนต่อท้ายออกก่อนบันทึกลงฐานข้อมูล เพื่อให้สอดคล้องกับคีย์นอก (Foreign Key) `menu_item_id` เสมอ บันทึกข้อมูลและปิดบิลได้ถูกต้อง 100%

### 4. แก้ไขบั๊กรูปภาพเมนูไม่บันทึก (Image URL Save Fix)
*   ปรับปรุง [App.js](file:///c:/dev/POS/src/js/core/App.js) เพิ่มเติมตัวแปร `image_url` ในคำสั่ง `insert` และ `update` ตาราง `menu_items` ช่วยแก้ปัญหาภาพที่อัปโหลดไม่ถูกบันทึกลงฐานข้อมูลเมนู ทำให้รูปภาพหน้า POS แสดงผลได้ถูกต้อง

### 5. ปรับการแสดงผลรูปภาพอาหารฝั่งลูกค้าให้เต็มกรอบ (Full-Frame Card Images)
*   ปรับปรุง [customer.css](file:///c:/dev/POS/src/css/customer.css) ในคลาส `.c-menu-card__img-container img` โดยเปลี่ยนจากรูปแบบเดิม (ที่จำกัดขนาดรูปไว้ที่ 90% ของกล่อง) ให้มีขนาดกว้างและสูงแบบ 100% พร้อมระบุ `object-fit: cover` เพื่อให้รูปภาพจริงขยายและครอบคลุมเต็มพื้นที่กรอบการ์ด ช่วยให้ดีไซน์อาหารหน้าเว็บฝั่งลูกค้าดูอิ่ม เต็มกรอบ และระดับพรีเมียมยิ่งขึ้นครับ

### 6. ระบบเพิ่ม/ลบหมวดหมู่ในหลังบ้าน (BOH Category Management Fix)
*   เชื่อมต่อคอลแบ็ก `onSaveCategoryCallback` และ `onDeleteCategoryCallback` ใน [App.js](file:///c:/dev/POS/src/js/core/App.js) เพื่อทำการบันทึกและอัปเดตสถานะหมวดหมู่สินค้าใน Supabase ได้จริง
*   เมื่อผู้ใช้ทำการเพิ่มหรือลบหมวดหมู่ (เช่น **Burritos** และ **Nachos**) ระบบจะรีโหลดข้อมูลจากดาต้าเบสแบบเรียลไทม์ และเปิดหน้าต่างจัดการหมวดหมู่ค้างไว้โดยอัตโนมัติ เพื่อมอบประสบการณ์การทำงานที่ลื่นไหลไม่ขัดจังหวะ

### 7. เครื่องมือจัดการราคาและตัวเลือกย่อยของเมนู (Variants Editor UI & Dynamic Saving)
*   เพิ่ม UI จัดการตัวเลือกย่อยสไตล์ iOS ในหน้ารายละเอียดเมนูฝั่งหลังบ้าน (BOH) เพื่อเปิดใช้งานและระบุตัวเลือกต่าง ๆ (เช่น ไก่, หมู, เนื้อ) พร้อมระบุราคาย่อยเฉพาะชิ้นได้อย่างอิสระ
*   เชื่อมโยงการบันทึกข้อมูลตัวเลือกย่อยในส่วนของการสร้างใหม่และแก้ไขเมนูเดิม โดยระบบจะทำการบันทึกลงในฟิลด์ `variants` ชนิดข้อมูล `JSONB` บน Supabase โดยอัตโนมัติ ทำให้ผู้ใช้แก้ไขและอัปเดตราคาของตัวเลือกย่อยแต่ละชนิดได้ทันทีผ่านระบบหน้าบ้าน

### 8. ระบบเลือกตัวเลือกอาหารฝั่งลูกค้า (Customer Portal Option Selection Support)
*   พัฒนาหน้าอาหารฝั่งลูกค้า [customerHome.js](file:///c:/dev/POS/src/js/pages/customerHome.js) ให้รองรับการเลือกตัวเลือกย่อยในหน้ารูปภาพและรายละเอียดเมนู (Food Detail Modal) แบบอัตโนมัติ
*   เมื่อกดเลือกเมนูที่มีตัวเลือกย่อย ระบบจะแสดงแถบตัวเลือกย่อยให้ลูกค้ากดเลือกพร้อมราคาที่ปรับเปลี่ยนตามแบบไดนามิก และบันทึกเข้าสู่ตะกร้าฝั่งลูกค้าในรูปแบบไอดีแยกย่อยตามชนิดวัตถุดิบ ทำให้หน้าร้าน POS ได้รับข้อมูลออเดอร์ย่อยได้อย่างถูกต้อง แม่นยำ และไม่มีข้อผิดพลาด

## ⚡ วิธีการปรับฐานข้อมูล & รันการนำเข้าเมนูย่อย (Database Migration & Menu Import)

เนื่องจากระบบมีการเพิ่มคอลัมน์ใหม่ในฐานข้อมูลออนไลน์ รบกวนทำตามขั้นตอนด้านล่างนี้เพื่อเปิดทำงานครับ:

### ขั้นตอนที่ 1: รัน SQL Command ในหน้า Supabase Dashboard
กรุณาเปิดหน้า **Supabase SQL Editor** บนคลาวด์ของคุณ แล้วคัดลอกคำสั่ง SQL นี้ไปวางและกด **Run**:
```sql
ALTER TABLE public.menu_items ADD COLUMN IF NOT EXISTS variants JSONB DEFAULT NULL;
```

### ขั้นตอนที่ 2: กดรันคำสั่งนำเข้าผ่าน Console หน้าร้าน
เมื่อรันคำสั่ง SQL ในขั้นตอนที่ 1 แล้ว:
1. รีเฟรชหน้าเว็บ POS (`http://localhost:3000/pos`)
2. กดปุ่ม **`F12`** เพื่อเข้าสู่แท็บ **Console**
3. รันโค้ดนำเข้าชุดใหม่นี้:
   ```javascript
   await window.importHandwrittenMenu()
   ```
